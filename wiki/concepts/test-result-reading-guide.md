# Test Result Reading Guide

Use this page as a **reading guide**, not as proof of final typing. In PsyCalc, a test result is a **pilot hypothesis** that still needs comparison with observation, self-description, and possible alternatives. See [[main-idea]] and [[compatibility-level-boundaries]].

## Level Separation

Read each system only at its own level:

- **Socionics** = **tactical** level only -> information modeling.
- **Psychosophy** = **operational** level only -> action organization.
- **Temporistics** = **strategic** level only -> temporal-existential orientation.

Do not treat one system as a translation of another. A result in one layer may clarify that layer without deciding the others.

## How to Read a Result

If a result shows a single best match plus alternatives, read it as a ranked hypothesis set:

- **Top-1** = current best fit inside that test model.
- **Top-3** = the three most plausible candidates worth reviewing together.
- Nearby candidates often mean the profile is not sharply separated yet.

In practice, compare the Top-3 pages and ask which description best explains repeated patterns over time, not just mood or recent stress.

## Clarity Labels

- **Low clarity** = weak separation between candidates. Treat the result as exploratory.
- **Moderate clarity** = usable working hypothesis, but still open to revision.
- **High clarity** = stronger signal inside this test, but still not final typing.

High clarity does **not** mean certainty. It only means the current test produced a cleaner internal pattern.

## Reading Composite Results

If you receive one result from each system, read them side by side but keep boundaries intact:

- Socionics result -> tactical information fit
- Psychosophy result -> operational action fit
- Temporistics result -> strategic time/meaning fit

The combined profile may be useful as a broader hypothesis, but it should not erase level boundaries.

## Practical Rule

Treat every test result as:

```text
test output -> provisional type hypothesis -> compare with Top-3 -> check clarity -> verify against repeated evidence
```

Prefer cautious wording: *likely*, *current best fit*, *working hypothesis*, *needs verification*.
