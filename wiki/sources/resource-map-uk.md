# Карта інтернет-ресурсів

Ця сторінка — кураторська карта інтернет-ресурсів, які можуть бути корисними для досліджень PsyCalc, поповнення джерел і відстеження походження тверджень.

Це **не** список визнаних авторитетів. Наявність ресурсу в карті означає лише: «цей ресурс варто перевірити». Будь-яке теоретичне твердження все одно потребує окремої перевірки перед використанням у wiki.

Див. також [[multilingual-translation-policy]], [[hypothesis-status-of-psycalc]] і [[limits-of-typological-inference]].

## Для чого потрібна карта

Карта допомагає:

- відрізняти першоджерела від переказів і коментарів;
- не втрачати нестабільні веб-матеріали;
- фіксувати мову, школу, контекст і застереження щодо надійності;
- спрямовувати майбутній ingest у `raw/`, `wiki/sources/` і похідні concept pages;
- відокремлювати інтерпретації PsyCalc від того, що справді стверджує зовнішнє джерело.

## Типи джерел

| Мітка | Значення | Типове використання |
|------|----------|---------------------|
| `primary-source` | Оригінальний текст автора, прямий архів школи, інтервʼю, первинна публікація | Базові теоретичні твердження |
| `secondary-summary` | Огляд, навчальне пояснення, стаття, переклад із коментарями | Орієнтація і порівняння |
| `critique` | Методологічна, емпірична, богословська, етична або скептична критика | Застереження і non-claims |
| `archive` | Дзеркало, web archive, репозиторій документів | Збереження і відновлення джерел |
| `community` | Форум, група, обговорення, неформальна практика типування | Слабке свідчення, варіанти термінів |
| `tool` | Тест, калькулятор, пошуковий індекс, датасет, утиліта | Практичні експерименти, не доказ |
| `project-hypothesis` | Внутрішній синтез або розширення PsyCalc | Не можна приписувати зовнішньому джерелу |
| `unverified` | Знайдено, але ще не перевірено вручну | Тільки черга на перевірку |

## Мітки надійності

Надійність відокремлюється від типу джерела.

| Мітка | Значення |
|------|----------|
| `high` | Першоджерело, стабільний архів, рецензована або добре документована публікація |
| `medium` | Відомий тематичний сайт з авторами або редактурою |
| `low` | Форум, анонімний блог, компіляція без посилань, AI-текст, нестабільний пост |
| `unknown` | Ще не перевірено |

## Мітки доступності

| Мітка | Значення |
|------|----------|
| `active` | URL працює і виглядає підтримуваним |
| `stale` | URL працює, але ресурс виглядає старим або непідтримуваним |
| `archived` | Основний доступ через архів або дзеркало |
| `dead` | Початковий URL не працює |
| `restricted` | Paywall, логін або інший обмежений доступ |
| `offline-copy` | Збережено локально або вручну, публічно недоступно |

### Богословʼя, етика і пастирські межі

Сюди заносяться ресурси, які допомагають відокремлювати типологічні евристики від християнської антропології, пастирського розсуду, моральної відповідальності та принципів формування сімʼї.

## Правила використання тверджень

1. Одного URL недостатньо. Важливі твердження потребують конкретної source page, цитати або близького переказу і дати доступу.
2. Не використовувати community-ресурси для базових теоретичних тверджень без сильнішого джерела.
3. Якщо PsyCalc розширює джерело, позначати розширення як `project-hypothesis`.
4. Якщо джерела розходяться, фіксувати розбіжність, а не склеювати її в хибний консенсус.
5. AI-зведення не є доказом. Вони можуть лише допомогти знайти джерела для людської перевірки.
6. Биті посилання краще позначати і замінювати архівами, а не мовчки видаляти.

## Ритм підтримки

- Додавати картку під час знаходження нового веб-ресурсу.
- Під час ingest переносити перевірені матеріали в `raw/` і створювати/оновлювати сторінку в `wiki/sources/`.
- Перевіряти важливі посилання раз на квартал або перед великою публікацією.
- Позначати мертві посилання як `dead` і додавати архівний URL, якщо він є.
- Тримати карту навігаційною; докладний аналіз виносити в окремі source pages.
